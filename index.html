The Async Execution Lifecycle

How SWASTHYA AI CORE manages complex research in the background.

Context Extraction

The Context API securely analyzes the incoming message stream to build a compliant `PatientContext` schema.

Discovery Initialization

If the context is sufficient, `POST /api/discovery/search` is invoked. A `task_id` is generated and returned immediately.

Background Execution

The internal Job Executor spawns a non-blocking coroutine, scraping NABH databases and Maps without holding the HTTP connection hostage.

Ranking & Explanation

Discovered hospitals are deterministically ranked. The LLM generates human-readable explanations summarizing the rationale.

Task Completion

Redis is updated to 100%. The client's polling mechanism captures the complete `RecommendationBundle` payload.

Modern Technology Stack

FastAPI Python 3.10+ Redis Gemini / Mistral Playwright Google Maps Docker AsyncIO

Built entirely on standard REST and async primitives. Zero bloat from heavy frameworks like LangChain. Dependency-isolated and optimized for Dockerized distribution.

Engineering Hard Problems

  • Context Versioning: Managing sliding Redis state across multi-turn sessions without polluting older confirmed data via Delta extraction.
  • Browser Pooling: Optimizing Playwright Chromium lifecycle in a constrained Docker container to minimize RAM overhead.
  • Token Minimization: Abstracting prompts securely to reduce token ingestion by 60%, drastically accelerating Time-To-First-Token (TTFT).
